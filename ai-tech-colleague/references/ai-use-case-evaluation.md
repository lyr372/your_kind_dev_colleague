# AI Use Case Evaluation

Use this to help a beginner judge whether an AI idea is worth trying.

## Quick Questions

Ask these in plain language:

- What does the user give the system? Text, document, picture, data table, or a business action?
- What should the system return? Answer, summary, category, extracted fields, draft, reminder, or decision?
- Does AI only help prepare something, or does it make the final decision?
- If AI is wrong, is it acceptable for a human to fix it?
- Is there a small set of examples we can test first?

## Good AI Fit

AI is usually useful when the task involves:

- Reading lots of text.
- Searching documents by meaning.
- Summarizing meetings, reports, tickets, or emails.
- Extracting fields from messy documents.
- Drafting replies, reports, or explanations.
- Classifying text into categories.

## Maybe Not AI First

AI may not be the first choice when:

- The task is just calculating numbers.
- The rule is fixed and clear.
- The system must be 100% correct without human review.
- The real problem is missing data or messy process ownership.
- The request is "make it smart" but no user or output is clear.

## Shrink The Idea

When a request is too big:

- "AI 助手回答所有问题" -> "先回答一种制度里的 20 个高频问题。"
- "AI 自动审批" -> "先做 AI 审核建议，人来点通过或驳回。"
- "AI 自动生成报告" -> "先用固定表格生成一版草稿。"
- "AI 读所有合同" -> "先抽一种合同里的 5 个字段。"
